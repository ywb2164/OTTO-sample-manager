# OTTO Sample Manager

一个面向鬼畜 / 人力调音工作流的桌面音频素材管理器。  
适用于 Melodyne（主）、Vegas、FL Studio 等场景，把常用素材的**导入、预览、整理、拖拽复用、预解码**，节省加载时间，并且有**一键活字印刷**功能，极大提升人力鬼畜的效率。

## 项目特色

制冰时对素材处理的常见问题：

- 素材散落在多个文件夹里，整理困难
- 导入后试听、比对和筛选效率低
- 频繁拖入 DAW / 调音软件时，工作流中断
- 重复使用素材时容易弄乱原始文件
- 一旦素材量变大，界面和响应速度很容易变差

OTTO Sample Manager：

- 保留原有素材目录结构
- 快速导入和批量管理采样
- 统一试听与波形预览
- 面向实际制作流程优化拖拽与复用体验
- 尽量减少重复解码、重复 IO 和无意义缓存

---

## 项目定位

核心优先级是：

1. **性能**：导入快、解码快、列表响应快  
2. **工作流**：方便试听、分组、拖拽、重复使用  
3. **界面**：保持简洁，轻量资源管理器

## 主要功能

### 1. 目录树导入
- 支持按目录导入音频素材
- 保留根目录结构
- 支持多个根目录叠加管理

---

### 2. 音频预览与波形显示
- 支持播放 / 暂停
- 底部播放器显示进度
- 支持拖动进度条 seek
- 内置波形缓存用于快速预览

---

### 3. 拖拽到调音软件
- 支持拖入 Melodyne、FL Studio 等软件
- 优化“试听 → 拖出”的连续操作体验

---

### 4. 自动副本机制（主要特点）
防止外部软件污染原素材（尤其是 Melodyne 对复用素材会一起操作）：

- 第一次拖出：使用原文件
- 第二次开始：生成副本
- 副本存储在 **应用安装位置 / Copy**
- 副本不在 UI 显示

---

### 5. 活字印刷生成（歌词自动组装 🔥）

核心新功能：**从歌词自动生成可用素材序列**

工作流：

```
歌词 txt → 拼音解析 → 匹配声库 → 自动生成新分组 → 有序音频素材
```

功能说明：

- 输入歌词 txt（多行文本）
- 自动逐字转为**无声调拼音**
- 选择一个已有分组作为“声库”（如：电棍 / 张顺飞等）
- 自动新建分组（默认使用 txt 文件名）
- 按歌词顺序匹配单字素材并复制
- 自动生成有序文件：

```
001_我_wo.wav
002_想_xiang.wav
003_要_yao.wav
...
```

#### 拼音匹配规则（已优化）

支持以下素材命名形式：

- `ba.wav`（标准拼音）
- `ba2.wav`（数字声调）
- `bà.wav / bá.wav`（带音调）

匹配逻辑：

1. 优先按汉字匹配  
2. 再按拼音匹配（统一归一化处理）  

归一化规则：

- 去数字：`ba2 → ba`
- 去音调：`bà → ba`
- 全部转小写

👉 因此三种形式都会匹配成功

#### 特点

- 自动生成完整素材序列（核心工作流）
- 重复候选素材：直接取第一个
- 缺失素材：不会中断，会统计提示
- 生成结果**永久保存**

#### 文件存储位置

生成素材存储在：

```
应用目录 / Copy / lyrics-assemblies /
```

说明：

- 与普通副本共享 Copy 体系
- 但**不会被清理**
- 属于长期资产（不是临时文件）

## 使用方法

### 1. 导入素材
导入你的音频目录（建议提前按项目 / 角色 / 音源整理好）

---

### 2. 浏览
按结构查看素材，支持批量选择

---

### 3. 试听
点击素材即可播放：

- 播放 / 暂停
- 查看进度
- 拖动 seek

---

### 4. 拖拽使用
直接拖入 Melodyne / FL Studio 使用

---

### 5. 重复使用保护
多次拖出自动使用副本，避免污染原文件

---

### 6. 活字印刷生成（歌词转素材）

步骤：

1. 点击顶部 **“活字印刷生成”**
2. 选择歌词 txt 文件
3. 选择声库分组（如“电棍”）
4. 确认生成分组名称
5. 自动生成素材序列

结果：

- 自动创建新分组
- 自动按顺序生成音频
- 可直接用于人力调音 / 拼接
