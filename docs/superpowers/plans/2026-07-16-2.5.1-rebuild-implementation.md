# OTTO Sample Manager 2.5.1 Rebuild Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a uniquely versioned 2.5.1 Windows installer, overwrite the stale installation at `E:\采样管理器`, and verify the installed GUI binary plus real file drag-out path.

**Architecture:** Keep the existing Tauri implementation, add a Node-based release artifact verifier that treats the compiled EXE and NSIS package as inputs, and make CI run the same verifier. Build from a clean release target, install silently to the existing location, then compare the installed binary with the verified build before launching it for desktop interaction testing.

**Tech Stack:** Tauri 2, Rust, NSIS, Node.js ESM, Vitest, PowerShell, Windows PE/COFF.

### Task 1: Add a release artifact verifier with a failing integration test

**Files:**
- Create: `scripts/verify-tauri-release.mjs`
- Create: `electron/main/services/tauriReleaseVerifier.test.ts`

- [ ] **Step 1: Write the failing test**

Create a temporary minimal PE file with Subsystem `3` and assert that invoking `node scripts/verify-tauri-release.mjs --exe <fixture> --installer <2.5.1 fixture> --expected-version 2.5.1` exits non-zero with `expected Windows GUI subsystem 2`. Add a second fixture with Subsystem `2` and assert that it exits zero and returns a JSON object containing SHA-256, version, subsystem, and installer path.

- [ ] **Step 2: Run the test to verify RED**

Run: `npm test -- electron/main/services/tauriReleaseVerifier.test.ts`

Expected: FAIL because `scripts/verify-tauri-release.mjs` does not exist.

- [ ] **Step 3: Implement the minimal verifier**

The script must:

```js
const peOffset = executable.readUInt32LE(0x3c)
const subsystem = executable.readUInt16LE(peOffset + 24 + 68)
if (subsystem !== 2) throw new Error(`expected Windows GUI subsystem 2, received ${subsystem}`)
```

It must also reject an installer whose basename does not contain the exact expected version, verify `package.json`, `src-tauri/tauri.conf.json`, and `src-tauri/Cargo.toml` all declare that version, and print a JSON verification report including both SHA-256 hashes.

- [ ] **Step 4: Run the focused test to verify GREEN**

Run: `npm test -- electron/main/services/tauriReleaseVerifier.test.ts`

Expected: 2 tests pass.
